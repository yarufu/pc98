## FM効果音仕様

ADV98.EXE では、PMD の FM効果音機能を使用します。

FM効果音データは、実行ファイルと同じフォルダに配置した `SE.EFC` から読み込みます。

`#se <番号>` で0～127のFM効果音を再生します。引数が範囲外、数値以外、
または `SE.EFC` が読み込まれていない場合は、何もせずゲームを続行します。

```
#se 2
```

効果音番号 1 はシステム効果音として使用されています。

## 背景表示

背景命令は、拡張子を含むG98ファイル名を直接指定します。
ファイル名はMS-DOSの8.3形式（ベース名8文字以内、拡張子3文字以内）とし、
ディレクトリや空白は使用しません。エンジンは`.G98`を自動付加しません。

`#bg <G98ファイル名>`は通常表示、
`#bginterlace <G98ファイル名>`は640×400全体をインターレース表示します。
背景の上側は偶数ラインを上から下へ、下側は奇数ラインを下から上へ、
1ラインおきに段階的に表示されます。背景名には通常のG98ファイルを指定します。

```
#bg CLASS.G98
#bginterlace CLASS.G98
```

引数なし、またはファイル読み込み失敗時は、黒画面に`BG LOAD NG`を表示して
ゲームを続行します。

演出後は、GameStateに保存されている現在の左右立ち絵を再描画します。
セーブ時は通常背景と同じくG98ファイル名だけを保存します。
ロード時は完成背景として復元され、インターレース演出は再実行されません。

## 立ち絵インターレース表示

立ち絵命令は、拡張子を含むSPRファイル名を直接指定します。
ファイル名はMS-DOSの8.3形式（ベース名8文字以内、拡張子3文字以内）とし、
ディレクトリは指定しません。エンジンは`.SPR`を自動付加しません。

通常の`#left <SPRファイル名|none>`と`#right <SPRファイル名|none>`で
立ち絵だけを部分更新する場合は、表示中のページ0を維持したまま、ページ1へ対象領域の
背景を読み込みます。SPR指定時はその上へ透明合成して完成矩形を作り、既存8KBバッファを
経由してページ0へ8ライン単位で転送します。各単位の書き込み順はB/R/G/Iです。

通常命令へ`none`を指定した場合は、ページ1へ背景だけの完成矩形を作り、同じ8ライン転送で
旧立ち絵を即時消去します。8ラインという転送単位は内部実装で固定されており、スクリプトから
指定または変更できません。

背景変更、ロード復帰、メニュー後などの全画面再描画では、背景全体の直後に左右SPRを
通常透明合成します。部分更新用の転送は再実行せず、背景矩形の再読み込みも行いません。
Hキーでは通常はメッセージ枠矩形だけを背景から復元し、枠が立ち絵領域と重なる場合、または
矩形復元に失敗した場合だけ、背景全体と左右SPRを完成状態で再描画します。

`#leftinterlace <SPRファイル名|none>` と
`#rightinterlace <SPRファイル名|none>` は、裏画面VRAMへ対象領域の背景を読み込み、
SPR指定時はその上へ透明合成した完成矩形を作ってから、ページ0へ走査線状に表示します。
表示ページはページ0のまま維持され、演出中はページ1からRAMを経由するVRAM転送だけを行います。

`none`を指定すると、裏画面には背景だけの完成矩形を作ります。その背景を同じ順序で
転送するため、表画面の旧立ち絵は一瞬で消えず、インターレース状に段階消去されます。
引数なしの命令は、状態と画面を変更せず無視します。背景またはSPRの準備に失敗した場合も
表画面は変更せず、表示ページとCPUアクセスページをページ0へ戻します。

通常表示の `#left` / `#right` とは別命令であり、転送するラインの意味と順序も異なります。

```
#left C01_N.SPR
#right C02_H.SPR
#leftinterlace MIO_UNI.SPR
#rightinterlace MIO_EV1.SPR
#left none
#right none
#leftinterlace none
#rightinterlace none
```

通常命令の`none`は即時消去し、インターレース命令の`none`は段階消去します。
従来のキャラクター番号と表情名の引数は廃止しました。
制服・私服・イベント専用画像などを、任意の8.3形式SPR名で指定できます。

GameStateには従来どおりSPRファイル名だけを保存します。ロード復帰、メニュー後の再描画、
Hキーのフォールバック再描画では完成画面を通常描画し、インターレース演出は再実行しません。
