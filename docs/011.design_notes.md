# 第24話：#rightscan2 演出調査

日付：2026/6/6

商用ゲーム風の立ち絵表示を目指して試作（インターレース演出させたい）

- 隙間埋め型インターレース
- 上下移動型インターレース

大工事になりそうなので本件見送り。

現時点では以下を正式採用する。

- #bgwipe
- #leftwipe
- #rightwipe

# 第25話：可変長 #choice 実装

日付：2026/6/8

## ia16-gcc 最適化回避

可変長 #choice 実装時、

- input_wait_choice_jis()
- handle_choice_block()

で ia16-elf-gcc の `-O2` による Internal Compiler Error (ICE) が発生した。

そのため現状は以下の関数のみ局所的に

```
__attribute__((optimize("O0")))
```

## マウス操作

日付：2026/6/11

### 選択メニューのマウス移動閾値

選択メニューではマウス移動量を十字キー入力へ変換している。

以下でマウス移動の閾値を設定
```
#define MOUSE_CHOICE_MOTION_THRESHOLD 16 ←やっぱ32から16にしました。
```

実機およびエミュレータで確認した結果、
- 8 では敏感すぎる
- 16 でもやや敏感
- 32 で市販ゲームに近い操作感


# 第26話：#call / #return を追加した理由

当初は #jump のみでシナリオを作成していた。

しかし、実際に「見る」「話す」「移動」を使った
コマンド選択式ADVのサンプルを書いたところ、

- 同じ説明文
- 同じ移動失敗メッセージ
- 同じ選択肢

を何度も記述する必要があった。

例：

- 「まだ帰る時間ではない」
- 「容疑者たちが今ここに集められている」

など。

スクリプト量が増えると保守が困難になるため、
共通処理をまとめる仕組みとして
#call / #return を追加した。

## 実装方針

- #call ラベル名
  - 現在位置をコールスタックへ保存
  - 指定ラベルへ移動

- #return
  - コールスタックから復帰

## 備考

- ネスト対応
- セーブ／ロード時もコールスタック保存
- CALL_STACK_MAX = 8
