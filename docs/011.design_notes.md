# 第24話：旧ワイプ演出の実装と削除

日付：2026/6/6、削除：2026/7/11

## 目的
商用ゲームでは背景やキャラの表示時に演出処理が入っているため、
インターレース完成前の代替として中央から上下へ開く演出を実装した。

## 結果

背景・左右立ち絵向けに実装したが、インターレース完成により役割が重複した。
コードセグメント節約のため、スクリプト命令、描画関数、専用helperを完全削除した。
現在の段階表示にはインターレース命令を使用する。

## その他

試作時は 2 ライン単位では遅すぎたため 16 ライン単位へ変更していた。
この調整値を含む専用実装は現在のソースには残していない。

# 第25話：可変長 #choice 実装

日付：2026/6/8

## ia16-gcc 最適化回避

可変長 #choice 実装時、

- input_wait_choice_jis()
- handle_choice_block()

で ia16-elf-gcc の `-O2` による Internal Compiler Error (ICE) が発生した。

そのため現状は以下の関数のみ局所的に

```
__attribute__((optimize("O0")))
```

### 選択メニューのマウス移動閾値

選択メニューではマウス移動量を十字キー入力へ変換している。

- 8 では敏感すぎる
- 16 でもやや敏感
- 32 で市販ゲームに近い操作感

# 第26話：#call / #return を追加した理由

当初は #jump のみでシナリオを作成していた。

しかし、実際に「見る」「話す」「移動」を使った
コマンド選択式ADVのサンプルを書いたところ、

- 同じ説明文
- 同じ移動失敗メッセージ
- 同じ選択肢

を何度も記述する必要があった。

例：

- 「まだ帰る時間ではない」
- 「容疑者たちが今ここに集められている」

など。

スクリプト量が増えると保守が困難になるため、
共通処理をまとめる仕組みとして
#call / #return を追加した。

## 実装方針

- #call ラベル名
  - 現在位置をコールスタックへ保存
  - 指定ラベルへ移動

- #return
  - コールスタックから復帰

## 備考

- ネスト対応
- セーブ／ロード時もコールスタック保存
- CALL_STACK_MAX = 8

# 第28話：FM効果音仕様

ADV98.EXE では、PMD の FM効果音機能を使用します。

FM効果音データは、実行ファイルと同じフォルダに配置した `SE.EFC` から読み込みます。

```
SE.EFC
```

PMD の仕様上、FM効果音番号は 0～127 を使用できます。

### システム効果音

ADV98.EXE では、効果音番号 1 をシステム効果音として予約しています。

1 : システム決定音

### SE.EFC が存在しない場合

SE.EFC が存在しない場合でも、ADV98.EXE はそのまま起動・動作を継続します。

この場合、

- FM効果音は再生されない
- BGM再生は通常どおり動作する
- タイトル画面は正常動作する
- セーブ／ロードは正常動作する
- スクリプト進行は正常動作する

という仕様です。

### 設計メモ：SE.EFC運用方針

ADV98.EXE では、FM効果音データを個別ファイルではなく、PMD標準形式の `SE.EFC` にまとめて管理します。

理由：

- PMD標準の運用方法に合わせるため
- 効果音管理を1ファイルに集約できるため
- 効果音番号で簡単に再生できるため
- `#se` 命令との相性が良いため

また、`SE.EFC` は必須ファイルではありません。

BGM再生とFM効果音は独立しているため、

```txt
PMD常駐
SE.EFCなし
```

現在は以下のタイミングで再生されます。

- タイトル画面の「はじめから」決定時
- タイトル画面の「ロード」決定時
- セーブ成功時

# 第30話：立ち絵インターレース表示

`#leftinterlace` / `#rightinterlace` を追加。

## インターレース演出

商用PC-98 ADV風の立ち絵表示演出。

当初は表示範囲を広げる方式を試したが、
参考ゲームとは見た目が異なったため不採用。

最終的には、

- 上側走査線群
- 下側走査線群

を独立して移動させる方式を採用。

走査線は2ライン周期、
32ライン帯を8ラインずつ移動させることで、
商用ゲームに近い印象を得られた。

大きなヒープ確保は行わず、
32ライン固定バッファを用いて実機でも高速に動作する。

# 第31話：立ち絵SPRファイル名の直接指定

立ち絵IDと表情名から固定ファイル名を作る方式を廃止し、スクリプトから
8.3形式のSPRファイル名を直接指定する方式へ変更した。

これにより、制服・私服・イベント専用画像を新しいenumや変換表なしで指定できる。
`none`は空文字列として保持し、立ち絵を非表示にする。セーブ形式はversion 3とし、
左右のSPRファイル名をそのまま保存する。version 2との変換処理は持たない。
