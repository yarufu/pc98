# 第26話：#call / #return を追加した理由

当初は #jump のみでシナリオを作成していた。

しかし、実際に「見る」「話す」「移動」を使った
コマンド選択式ADVのサンプルを書いたところ、

- 同じ説明文
- 同じ移動失敗メッセージ
- 同じ選択肢

を何度も記述する必要があった。

例：

- 「まだ帰る時間ではない」
- 「容疑者たちが今ここに集められている」

など。

スクリプト量が増えると保守が困難になるため、
共通処理をまとめる仕組みとして
#call / #return を追加した。

## 実装方針

- #call ラベル名
  - 現在位置をコールスタックへ保存
  - 指定ラベルへ移動

- #return
  - コールスタックから復帰

## 備考

- ネスト対応
- セーブ／ロード時もコールスタック保存
- CALL_STACK_MAX = 8

# 第28話：FM効果音仕様

ADV98.EXE では、PMD の FM効果音機能を使用します。

FM効果音データは、実行ファイルと同じフォルダに配置した `SE.EFC` から読み込みます。

```
SE.EFC
```

PMD の仕様上、FM効果音番号は 0～127 を使用できます。

### システム効果音

ADV98.EXE では、効果音番号 1 をシステム効果音として予約しています。

1 : システム決定音

現在は以下のタイミングで再生されます。

- タイトル画面の「はじめから」決定時
- タイトル画面の「ロード」決定時
- セーブ成功時

### SE.EFC が存在しない場合

SE.EFC が存在しない場合でも、ADV98.EXE はそのまま起動・動作を継続します。

この場合、

- FM効果音は再生されない
- BGM再生は通常どおり動作する
- タイトル画面は正常動作する
- セーブ／ロードは正常動作する
- スクリプト進行は正常動作する

という仕様です。

### 設計メモ：SE.EFC運用方針

ADV98.EXE では、FM効果音データを個別ファイルではなく、PMD標準形式の `SE.EFC` にまとめて管理します。

理由：

- PMD標準の運用方法に合わせるため
- 効果音管理を1ファイルに集約できるため
- 効果音番号で簡単に再生できるため
- 将来的な `#se` 命令との相性が良いため

また、`SE.EFC` は必須ファイルではありません。
