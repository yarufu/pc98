# ADV98 セーブシステム仕様

## 概要

ADV98 ではゲームの進行状況をセーブファイルへ保存できます。

セーブデータは 3 スロット対応です。

```txt
SAVE1.DAT
SAVE2.DAT
SAVE3.DAT
```

セーブおよびロードはシステムメニューから行います。

---

## セーブ対象

セーブ時には以下の情報を保存します。

* 現在のスクリプト位置
* 現在の背景
* 左立ち絵
* 左立ち絵の表情
* 右立ち絵
* 右立ち絵の表情
* 現在のBGM
* 通常フラグ
* 数値フラグ

ロード時にはこれらの状態を復元し、保存位置からゲームを再開します。

---

## セーブファイル形式

### SaveData

```c
typedef struct {
    char magic[8];
    int version;

    GameState state;
    GameFlag flags[MAX_FLAGS];
} SaveData;
```

---

### magic

セーブデータ識別子です。

```txt
ADV98SAV
```

セーブファイルが ADV98 用であることを判定します。

---

### version

セーブデータ形式のバージョン番号です。

保存時と読込時でバージョンが一致しない場合はロードに失敗します。

---

## GameState

ゲームの進行状態を保存します。

```c
typedef struct {
    char bg_name[32];
    int script_line;

    enum StandId left_stand;
    enum FaceId left_face;

    enum StandId right_stand;
    enum FaceId right_face;

    char bgm[64];
} GameState;
```

---

### bg_name

現在表示中の背景名です。

例：

```txt
school
classroom
gym
```

---

### script_line

現在実行中のスクリプト行番号です。

ロード時はこの位置から再開します。

---

### left_stand / left_face

左側キャラクターの立ち絵と表情です。

---

### right_stand / right_face

右側キャラクターの立ち絵と表情です。

---

### bgm

現在再生中のBGMファイル名です。

例：

```txt
opening.m
school.m
```

ロード後は自動的に再生を再開します。

---

## GameFlag

ゲーム内フラグを保存します。

```c
typedef struct {
    char name[32];
    int value;
} GameFlag;
```

通常フラグと数値フラグを共通形式で管理します。

例：

```txt
route_a = 1
money = 1000
love = 50
```

---

## ロード処理

ロード成功時は以下を実行します。

1. GameState を復元
2. GameFlag を復元
3. 背景を再描画
4. 立ち絵を再描画
5. BGM を再開
6. 保存位置からスクリプトを再開

---

## エラー処理

以下の場合はロードに失敗します。

* セーブファイルが存在しない
* セーブファイルを開けない
* magic が一致しない
* version が一致しない
* 読み込みサイズが不正

ロード失敗時はゲーム状態を変更しません。

---

## 備考

セーブデータ形式を変更した場合は version を更新してください。

version が異なるセーブデータとの互換性は保証されません。
